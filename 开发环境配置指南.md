## 一、快速开始

### 1.1 环境准备

#### 安装Python
1. 访问 [Python官网](https://www.python.org/downloads/)
2. 下载Python 3.10或更高版本
3. 安装时勾选 "Add Python to PATH"

#### 验证安装
```bash
python --version
# 应该显示 Python 3.10.x 或更高版本

pip --version
# 应该显示 pip 21.x 或更高版本
```

### 1.2 项目设置

#### 克隆项目
```bash
git clone <项目仓库地址>
cd 钻孔数据工具
```

#### 创建虚拟环境（推荐）
```bash
# Windows
python -m venv venv
venv\Scripts\activate

# Linux/Mac
python3 -m venv venv
source venv/bin/activate
```

#### 安装开发依赖
```bash
pip install -e ".[dev]"
```

或者手动安装：
```bash
pip install pytest pytest-cov ruff mypy bandit pre-commit
```

## 三、代码质量工具

### 3.1 Ruff（代码风格检查）

#### 配置文件
`ruff.toml` 已创建，包含以下规则：
- **E/W**: pycodestyle错误和警告
- **F**: pyflakes
- **I**: isort导入排序
- **N**: pep8-naming命名规范
- **UP**: pyupgrade代码升级
- **B**: flake8-bugbear常见问题
- **SIM**: flake8-simplify简化建议

#### 使用命令
```bash
# 检查代码风格
ruff check .

# 自动修复可修复的问题
ruff check --fix .

# 格式化代码
ruff format .

# 检查单个文件
ruff check borehole_app/parser.py
```

### 3.2 Mypy（类型检查）

#### 配置文件
`mypy.ini` 已创建，启用严格模式。

#### 使用命令
```bash
# 检查整个项目
mypy borehole_app

# 检查单个文件
mypy borehole_app/parser.py

# 生成类型检查报告
mypy borehole_app --html-report mypy-report
```

#### 常见类型错误修复
```python
# 错误：缺少类型注解
def process_data(data):
    pass

# 正确：添加类型注解
def process_data(data: list[str]) -> dict[str, int]:
    pass

# 错误：使用旧式类型注解
from typing import List, Dict

# 正确：使用Python 3.10+原生语法
def process(data: list[str]) -> dict[str, int]:
    pass
```

### 3.3 Pytest（测试框架）

#### 配置文件
`pyproject.toml` 中已配置pytest。

#### 使用命令
```bash
# 运行所有测试
pytest

# 运行特定测试文件
pytest tests/test_parser.py

# 运行特定测试类
pytest tests/test_parser.py::TestParseMainFile

# 运行特定测试方法
pytest tests/test_parser.py::TestParseMainFile::test_parse_normal_main_file

# 生成覆盖率报告
pytest --cov=borehole_app --cov-report=html

# 显示详细输出
pytest -v

# 显示失败测试的详细信息
pytest --tb=short

# 只运行标记为slow的测试
pytest -m slow

# 排除标记为slow的测试
pytest -m "not slow"
```

#### 编写测试
```python
import pytest
from pathlib import Path
from borehole_app.parser import parse_main_file

class TestParseMainFile:
    """测试parse_main_file函数。"""
    
    def test_parse_normal_file(self, tmp_path: Path) -> None:
        """测试解析正常文件。"""
        test_file = tmp_path / "ZK1"
        test_file.write_text("ZK1\n10\n★\n", encoding="utf-8")
        
        result = parse_main_file(test_file)
        
        assert result.lines[0] == "ZK1"
        assert result.lines[1] == "10"
    
    def test_parse_empty_file(self, tmp_path: Path) -> None:
        """测试解析空文件。"""
        test_file = tmp_path / "ZK1"
        test_file.write_text("", encoding="utf-8")
        
        result = parse_main_file(test_file)
        
        assert len(result.lines) == 16
        assert all(line == "" for line in result.lines)
    
    @pytest.mark.parametrize("depth,expected", [
        ("10", (0, 10.0)),
        ("abc", (1, "abc")),
        ("0", (0, 0.0)),
    ])
    def test_depth_key(self, depth: str, expected: tuple) -> None:
        """测试_depth_key函数。"""
        from borehole_app.parser import _depth_key
        assert _depth_key(depth) == expected
```

## 四、Git工作流

### 4.1 分支策略
- **main**: 生产环境代码
- **develop**: 开发分支
- **feature/***: 功能分支
- **bugfix/***: 修复分支
- **release/***: 发布分支

### 4.2 提交规范
使用约定式提交（Conventional Commits）：

```
<type>(<scope>): <subject>

<body>

<footer>
```

**类型**：
- `feat`: 新功能
- `fix`: 修复bug
- `docs`: 文档更新
- `style`: 代码风格调整
- `refactor`: 重构
- `test`: 测试相关
- `chore`: 构建/工具相关

**示例**：
```
feat(parser): 添加对新文件格式的支持

- 新增parse_new_format函数
- 更新相关测试用例
- 更新文档

Closes #123
```

### 4.3 代码审查流程
1. 创建功能分支
2. 完成开发并提交
3. 创建Pull Request
4. 自动运行CI检查
5. 至少一人审查代码
6. 审查通过后合并

## 五、常见问题解决

### 5.1 类型检查错误

**问题**：`error: Function is missing a type annotation`

**解决**：
```python
# 错误
def process(data):
    pass

# 正确
def process(data: list[str]) -> dict[str, int]:
    pass
```

### 5.2 导入排序错误

**问题**：`I001 Import block is un-sorted`

**解决**：
```bash
# 自动修复
ruff check --fix .
```

### 5.3 测试失败

**问题**：测试因文件路径问题失败

**解决**：
```python
# 使用tmp_path fixture
def test_something(tmp_path: Path) -> None:
    test_file = tmp_path / "test.txt"
    test_file.write_text("content", encoding="utf-8")
    
    result = process_file(test_file)
    assert result == expected
```

### 5.4 编码问题

**问题**：读取GBK文件时出现编码错误

**解决**：
```python
# 使用项目中的read_text_file函数
from borehole_app.parser import read_text_file

content = read_text_file(file_path)
```

## 六、最佳实践

### 6.1 代码风格
1. 使用4个空格缩进
2. 行长度限制88字符
3. 使用双引号字符串
4. 导入顺序：标准库 → 第三方库 → 本地模块

### 6.2 类型注解
1. 所有函数参数和返回值添加类型注解
2. 使用Python 3.10+原生语法
3. 复杂类型使用TypeAlias

### 6.3 文档字符串
1. 所有公开函数、类、模块添加文档
2. 使用Google风格docstring
3. 包含参数说明、返回值、异常情况

### 6.4 测试
1. 每个函数至少一个测试用例
2. 测试边界条件和异常情况
3. 使用pytest fixtures减少重复代码
4. 测试覆盖率目标80%+

### 6.5 错误处理
1. 使用具体的异常类型
2. 提供有用的错误信息
3. 记录日志便于调试
