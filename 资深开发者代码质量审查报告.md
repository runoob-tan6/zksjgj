## 二、本次修复的 Bug

### Bug 1: `parse_h_file` 空描述解析错误 (严重)

**文件**: `borehole_app/parser.py` 第 185-196 行

**问题**: 当岩性描述文件中某深度的描述为空，且下一行是另一个 `#深度` 时，解析器会将下一个 `#深度` 行错误地当作前一个深度的描述。

**复现数据**:
```
#5
(空行)
#10
描述2
```

**期望结果**: `{"5": "", "10": "描述2"}`
**实际结果**: `{"5": "#10"}`

**修复方案**: 添加判断 — 如果下一行以 `#` 开头，则当前深度的描述为空。

```python
# 修复前
if index + 1 < len(lines):
    description = lines[index + 1].strip()
result[depth] = description
index += 2

# 修复后
if index + 1 < len(lines) and not lines[index + 1].strip().startswith("#"):
    description = lines[index + 1].strip()
    index += 2
else:
    index += 1
result[depth] = description
```

### Bug 2: `writer.py` 缺少 `Iterable` 导入 (中等)

**文件**: `borehole_app/writer.py` 第 68 行

**问题**: `make_file_text()` 函数参数类型注解使用了 `Iterable[str]`，但未导入该类型。

**修复**: 添加 `from collections.abc import Iterable`。

### Bug 3: `test_file_lines` 命名冲突 (中等)

**文件**: `borehole_app/writer.py` 第 241 行

**问题**: 函数名 `test_file_lines` 以 `test_` 开头，pytest 会将其误当作测试用例收集，导致 fixture 错误。

**修复**: 重命名为 `build_test_file_lines`，并更新所有引用。

### 6.2 错误处理

```python
# ✅ 推荐: 结构化错误处理
import logging
logger = logging.getLogger(__name__)

def load_project(folder: Path) -> ProjectData:
    try:
        project = ProjectData(folder=folder)
        # ... 加载逻辑
        return project
    except FileNotFoundError as e:
        logger.error("项目文件夹不存在: %s", folder, exc_info=True)
        project.load_error = f"项目文件夹不存在: {folder}"
        return project
    except Exception as e:
        logger.error("加载项目失败: %s", e, exc_info=True)
        raise
```
